# Epic 5: Mission Control — Painel de Comando do Squad Playground

## Status
Ready

## Visao

Transformar o Squad Playground de uma ferramenta de execucao de pipeline em um **Mission Control completo**, onde Sancho tem visibilidade total sobre equipe, projetos, tarefas, conteudo e historico — tudo integrado ao fluxo de agentes que ja existe.

## Contexto

O Squad Playground ja possui:
- Pipeline de consultoria com 6 agentes sequenciais
- Escritorio digital com avatares pixelados e status visual (AgentHouse + GameCanvas)
- Sistema de aprovacao com preview de artefatos
- MicroTasks com parsing de [TASKS] em tempo real
- Artefatos salvos em `docs/artifacts/{sessionId}/`
- Configuracoes persistidas em localStorage

**O que falta**: visao consolidada de projetos, equipe, tarefas e conteudo.

## Ondas de Execucao

### Wave 1 — Quick Wins (Stories 5.1 + 5.6)
- Estrutura de equipe: componente novo, dados ja existem em AGENT_DEFINITIONS
- Refinamentos do escritorio: melhorias incrementais no que ja funciona
- **Dependencia**: Nenhuma

### Wave 2 — Core Features (Stories 5.2 + 5.3)
- Task Board: evolucao do MicroTasks com persistencia
- Historico de Projetos: listar sessoes e artefatos do filesystem
- **Dependencia**: Wave 1 (para navegacao coesa)

### Wave 3 — Advanced Features (Stories 5.4 + 5.5)
- Pipeline interativo: editor rico no ApprovalPopup
- Tela de memoria: indexacao full-text
- **Dependencia**: Wave 2 (historico alimenta memoria)

## Decisoes Arquiteturais

1. **Navegacao**: Adicionar sistema de rotas/tabs no App.tsx (sem React Router — tabs internas)
2. **Persistencia**: Filesystem para artefatos (ja existe), localStorage para estado do usuario
3. **Backend**: Novas rotas Express para listar sessoes e buscar artefatos
4. **Sem banco de dados**: Manter filesystem-based por enquanto (simplicidade)
